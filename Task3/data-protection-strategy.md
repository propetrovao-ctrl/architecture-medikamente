
## 1. Данные, обоснование защиты и инструменты

| Данные | В чём необходимость защиты | Инструменты защиты |
|--------|---------------------------|-------------------|
| PII | Закон о ПДн, идентификация при утечке | Шифрование, TLS, RBAC/ABAC, маскирование, теги |
| PHI | Медтайна, высокий риск | Отдельное хранилище, TLS, узкий API, журнал доступа |
| FIN | Налоги, мошенничество | Шифрование, TLS к 1С/ККМ, роли касса/бух, аудит |
| Логи | Косвенно раскрывают PHI | Минимизация, псевдонимы, отдельный контур, SIEM по ролям |
| Бэкапы | Полный срез ПДн | Шифрование, изоляция, отдельные учётки |
| Аналитика | Дублирование ПДн без обезличивания | ETL, обезличивание, озеро, политики каталога |

## 2. Шифрование по состояниям

| Состояние | Цель | Типовые меры |
|-----------|------|--------------|
| Покой | Нет читаемых данных при краже носителя | Диски, БД, бэкапы; ключи в KMS/HSM |
| Передача | Нет утечки по каналу | TLS, VPN при необходимости |
| Использование | Меньше утечки с ПК | Короткие сессии, MFA, DLP по политике |

## 3. Автоматизированный контроль

| Направление | Суть |
|-------------|------|
| Целевое ТЗ | Автопроверка релизов на теги, мониторинг и алерты, аудит доступа, нетипичные действия, оповещения |
| PbD, lineage, теги | Постоянный мониторинг мер; логи для проверки обработки ПДн; контроль lineage и полноты тегов |
| Потоки | Мониторинг потоков, алерты, lineage для разборов |
| Наблюдаемость | Метрики и логи (Prometheus, Grafana, ELK и др., Victoria Metrics), алерты; единый стек и ответственность за observability |
