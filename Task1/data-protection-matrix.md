Данные для защиты и способы защиты

PHI - шифрование в покое + строгий доступ + аудит
PII - доступ по ролям + маскирование в интерфейсах
FIN - доступ бухгалтерии/кассам + аудит

1. Персональные данные пациента: ФИО, дата рождения, телефон, email (PII).
Excel со списком пациентов, журналы записи, сканы договоров, почта.
Теги для учёта: PII, что это пациент.
Защита: на диске шифровать том/папки/бэкапы. По сети только защищённый доступ (TLS, VPN), не слать как есть куда попало. Кто что видит — по ролям. Для отчётов можно маскировать (например только последние цифры телефона).

2. Расширенная анкета: адрес, работа/учёба, хронические болезни.
Это и PII, и по сути медицинская чувствительность (PHI).
Теги: PII, PHI, высокий риск.
Защита: собирать только то, что реально нужно для услуги. Хранить и передавать с шифрованием. Для аналитики лучше обезличить. Доступ узкий: врачам больше, ресепшену меньше.

3. Медкарта, заключения, диагнозы (медтайна, PHI).
Теги: PHI, медицинская тайна.
Защита: отдельное хранилище, шифрование, бэкапы. По сети TLS. В работе — только тем, кому нужно по работе, с журналом кто смотрел. Копии для разработки и тестов — маскированные или выдуманные данные, не настоящие карты.

4. Результаты анализов и реестры по дням (Excel RegistryByDate и файлы).
Теги: PHI, анализы.
Защита: при обмене с лабораторией по API отдавать минимум полей. Во внешний мир лучше внутренний идентификатор или токен, а не полный паспорт в каждом сообщении. Шифрование при передаче и при хранении. Вести понятную цепочку: откуда пришло, что меняли (lineage).

5. Запись на приём, расписание, факт визита.
Теги: PII, запись, конфиденциально.
Защита: шифрование на диске и в канале. Врач видит своё, ресепшен общую запись, пациент только своё. В календарь не показывать лишние медицинские подробности, если они не нужны для записи.

6. Оплаты, чеки, бухгалтерия (FIN).
Теги: FIN, бухучёт.
Защита: шифрование, доступ только кассирам и бухгалтерии. В экранах показывать не всё подряд, а столько, сколько нужно для роли. Писать, кто создал или менял документы.

7. Склад, ТМЦ.
Могут быть цены и поставщики (внутренняя/коммерческая тайна).
Теги: внутреннее, при необходимости коммерческая тайна.
Защита: доступ склад + бухгалтерия по необходимости. Журнал изменений, чтобы было видно, кто что менял.
